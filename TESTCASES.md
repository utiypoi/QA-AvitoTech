
## Задание 2.1

API содержит 4 ручки:
- Создать объявления (POST /api/1/item)
- Получить объявления по его идентификатору (GET /api/1/item/{id})
- Получить все объявления по идентификатору продавца (GET /api/1/{sellerId}/item)
- Получить статистику по айтем id (GET /api/1/statistic/{itemId})

### Тестовые данные:
- sellerId: уникальный int в диапазоне 111111–999999
- name: строка
- price: целое число
- statistics: likes, contacts, viewCount: int

### 1. Создание объявления
|ID|Название|Шаги|Ожидаемый результат|
|-|-|-|-|
|TC1|Создание валидного объявления|1.Отправить POST /api/1/item c JSON   2.Проверить статус и тело ответа| HTTP 200, в ответе поле status содержит UUID созданного объявления|
|TC2|Создание объявления без поля name|1.Отправить POST без поля name| HTTP 400, сообщение о валидации|
|TC3|Создание объявления без полей sellerId|1.Отправить POST без sellerId|HTTP 400|
|TC4|Создание объявления при price=0|1.Отправить POST с price=0|HTTP 400 или 422|
|TC5|Создание объявления с sellerId вне диапазона 111111-999999|1.Отправить POST с sellerId=123|HTTP 400|
|TC6|Передать statistics как пустой объект|1. Отправить POST с statistics={}|HTTP 400|
### 2. Получить объявление по ID
|ID|Название|Шаги|Ожидаемый результат|
|-|-|-|-|
|TC7|Получение существующего объявления|1.Выполнить GET /api/1/item/{id} c id реального объявления   2.Проверить тело ответа| HTTP 200, массив из одного элемента, поля id, sellerId, name, price, statistics.*, createdAt not null|
|TC8|Получение объявления по несуществующему id| 1.Выполнить GET /api/1/item/00000000-0000-0000-0000-000000000000|HTTP 404|
|TC9|Получение объявления по id в неверном формате|1. Выполнить GET /api/1/item/123|HTTP 400 или 404|
### 3. Получить все объявления по ID продавца
|ID|Название|Шаги|Ожидаемый результат|
|-|-|-|-|
|TC10|Получить все объявления существующего продавца|1.Выполнить GET /api/1/{sellerId}/item|HTTP 200, массив содержит 2 и более элементов, sellerId равен ожидаемому значению|
|TC11|Получить объявления для продавца, у которого нет объявлений|1.Выполнить GET для sellerId, неиспользуемый раннее|HTTP 200, пустой массив|
|TC12|Получить все объявления для sellerId вне диапазона|1.Выполнить GET /api/1/123/item|HTTP 400|
### 4. Получить статистику по объявлению
|ID|Название|Шаги|Ожидаемый результат|
|-|-|-|-|
|TC13|Получение статистики существующего объявления|1.Выполнить GET /api/1/statistic/{id} для реального id  2.Проверить поля|HTTP 200, массив из одного элемента, поля likes, viewCount, contacts not null|
|TC14|Получение статистики по несуществующему id|1. Выполнить GET /api/1/statistic/{fakeId}|HTTP 404|
|TC15|Получение статистики по id неверного формата|1.Выполнить GET /api/1/statistic/12345|HTTP 400 или 404|
