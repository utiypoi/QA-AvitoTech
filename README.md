# QA-AvitoTech
Проект содержит тест-кейсы и авто тесты для проверки API микросервиса.  
Тестирование API по адресу: https://qa-internship.avito.com   

Микросервис включает 4 ручки:  
POST /api/1/item — создать объявление  
GET /api/1/item/{id} — получить объявление по его идентификатору  
GET /api/1/{sellerId}/item — получить все объявления продавца  
GET /api/1/statistic/{itemId} — получить статистику объявления    
У объявления должен быть уникальный sellerId (диапазон 111111–999999).   
У созданных объявлений отдаётся уникальный UUID.   

Структура репозитория:   
BUGS.md - баг-репорты по результатам тестирования   
TESTCASES.md - тест-кейсы   
Task1.md - решение задачи 1   
README.md - инструкция   
pom.xml - Maven-конфигурация   
src/ - автоматизированные API-тесты   

Запуск:   
1. Клонировать репозиторий
2. Проверить установку и доступность Maven
3. Запуск тестов

Автоматизированы тесты:

TC1 - создание валидного объявления

TC2 - отсутствие поля name

TC3 - отсутствие sellerId

TC7 - получение существующего объявления

TC8 - запрос несуществующего id

TC10 - получение всех объявлений по sellerId

TC11 - продавец без объявлений

TC13 - получение статистики по ID

TC14 - статистика по несуществующему ID

TC16 - price строкой

TC18 - отсутствие statistics   
Остальные тесты в TESTCASES.md - ручные.
